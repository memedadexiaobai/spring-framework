
ConcurrentReferenceHashMap初始化：(int) ((initialCapacity + size - 1L) / size)

这是 **“向上取整除法”** 的整数技巧写法，等价于  
`ceil((double) initialCapacity / size)`，  
**但全程用整数运算，避免浮点，更快、更精确。**

---

### 1  数学原理

对于正整数 `a, b`：

```
(a + b - 1) / b   ≡   ⌈a / b⌉
```

| 例子 | 普通除 | 向上取整公式 | 结果 |
|---|---|---|---|
| `a=10, b=3` | `10/3=3` | `(10+3-1)/3=12/3=4` | **4** |
| `a=9, b=3` | `9/3=3` | `(9+3-1)/3=11/3=3` | **3** |

**余数为 0 时不变，有余数时自动加 1。**

---

### 2  在本构造函数里的用途

```java
int roundedUpSegmentCapacity = (int) ((initialCapacity + size - 1L) / size);
```

- `initialCapacity`：用户希望的总初始容量
- `size`：已算出的分段数（Segment 数量）
- 结果：***每段**至少要分配多少槽位，才能保证 **总槽位 ≥ initialCapacity**（向上对齐）*

---

### 3  为什么用 **long 中间运算**？

```java
...+ size - 1L) / size
```

- 防止 `initialCapacity + size - 1` **超过 int 范围**出现负数；
- 一次强转 `long` 即可，最后结果再转回 `int`（Segment 容量一定是 int）。

---

### 4  一句话总结

> **“(n + d - 1) / d” 是整数向上取整的经典位运算友好写法：余数非零自动进 1，保证总容量不小于用户期望，且全程无浮点。**
